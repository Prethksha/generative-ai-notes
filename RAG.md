# Retrieval-Augmented Generation (RAG)
## What is RAG?
Retrieval augmented generation, or RAG, is an architecture for optimizing the performance of an artificial intelligence (AI) model by connecting it with external knowledge bases. RAG helps large language models (LLMs) deliver more relevant responses at a higher quality.
RAG combines:
1. Information Retrieval
2. Generative AI

## Workflow
User Query
→ Retrieve Relevant Documents
→ Send Context to LLM
→ Generate Accurate Response

## Benefits
The primary benefits of RAG include:

- Cost-efficient AI implementation and AI scaling
- Access to current domain-specific data
- Lower risk of AI hallucinations
- Increased user trust
- Expanded use cases
- Enhanced developer control and model maintenance
- Greater data security
- Better Accuracy
- Reduced Hallucinations
- Up-to-date Information

## Key concepts

1. RAG systems enable external knowledge sources to retrieve up-to-date information for greater accuracy and relevance, enhancing tasks such as customer support, research, content creation, and integration of domain-specific and proprietary data.

2. RAG expands an LLM’s knowledge base without frequent retraining. This supports cost-efficient AI scaling, increases user trust, and enhances data security, making AI applications more reliable and effective.

3. The RAG workflow includes five key steps: prompt creation, information search, information retrieval, prompt augmentation, and response generation. This keeps AI outputs current, relevant, and actionable, improving usability across real-world applications.

4. Integrating IBM Watson Discovery with RAG allows an HR virtual assistant to provide accurate, up-to-date policy information. This prevents misinformation, enhances trust, and provides reliable real-time responses for employees.
