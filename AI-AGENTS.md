## Key concepts

1. An AI agent is a system or program that is capable of autonomously performing tasks on behalf of a user or another system by designing its workflow and using available tools.

2. AI agents perceive, reason, act, and learn autonomously. This enables them to operate efficiently in dynamic environments by processing inputs, planning actions, and adapting based on new data and outcomes.

3. AI agent architectures are categorized as reactive, deliberative, or hybrid, based on how agents operate. Additionally, AI agents are classified into five types: simple reflex, model-based, goal-based, utility-based, and adaptive. Each type of AI agent is suited to a specific decision-making need and environment.

4. Structuring an AI agent’s workflow involves key steps that allow the agent to act coherently and efficiently. These key steps are identifying a goal, breaking the goal into tasks and subtasks, defining input data and decision points, connecting to systems, and specifying outputs.

5. Building a personalized AI health agent using IBM watsonx Agent Lab enables practical application of low-code development, tool integration, and agent workflow configuration without requiring extensive programming expertise.

6. When designing AI agents, it’s important to follow the best practices that support fairness, transparency, explainability, robustness, and privacy. These best practices help reduce risks and promote the ethical use of AI agents.

7. Evaluating AI agents involves selecting appropriate metrics, such as task adherence, tool use, output quality, efficiency, and ethical alignment, to help agents perform as intended and deliver value.

8. The AI agent deployment lifecycle consists of three distinct phases: predeployment, deployment, and postdeployment. In predeployment, an agent is prepared and tested for secure, seamless integration. Deployment involves transitioning an agent into operational systems for real-world use. Postdeployment focuses on agent monitoring, human oversight, and updates to maintain accuracy, reliability, and alignment with business needs.

9. Evaluating the performance of an AI agent using IBM Granite and MLflow helps test whether it’s ready for full-scale deployment.
